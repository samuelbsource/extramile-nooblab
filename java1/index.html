Variable scope

The scope of a variable is the part of the program where the variable is accessible. In Java, there are three levels of scope:

  • Class level: A variable that is defined at the class level is accessible from anywhere within the class - it is called a property of that class.
  • Method level: A variable that is defined within a method is only accessible within that method.
  • Block level: A variable that is defined within a block of code is only accessible within that block of code. (this is not relevant for this exercise)
To illustrate this, consider the following code: 
public class Main
{
    // This is a property
    public static String name;

    public static void main(String[] args)
    {
        // This is a method level variable
        String localName = "Alice";

        // This is a block level variable
        if (true) {
            String blockName = "Bob";
        }

        // This is a property
        name = "Charlie";

        // This works
        System.out.println(localName);

        // This works
        System.out.println(name);

        // This does not work
        System.out.println(blockName);

        printName();
    }

    public static void printName()
    {
        // This works
        System.out.println(name);

        // This does not work
        System.out.println(localName);

        // This does not work
        System.out.println(blockName);
    }
}

One more thing

You can use static on a property in Java just like you can use static on a method. A static property is bound to the class, not to an object of the class, you don't need to create an object to access a static property. Have a look at the following code to see what I mean: 

public class Main
{
    public static void main(String[] args)
    {
        // Accessing a static property
        System.out.println("Number of students: " + Student.studentCount);
        
        Student student1 = new Student("Alice");
        Student student2 = new Student("Bob");
        
        // Accesing a static property
        System.out.println("Number of students: " + Student.studentCount);

        // This works
        System.out.println("Student 1 name: " + student1.name);

        // This does not work
        System.out.println(Student.name);

        // You can do this, but it will cause the program to run in an infinite loop
        Main.main(args);
    }
}
public class Student
{
    // This is a static property
    public static int studentCount = 0;
    public String name;

    public Student(String name)
    {
        this.name = name;
        Student.studentCount++;
    }
}

The plan

Planning the functionality

The plan is to create a menu that looks like this: 

==== Menu ====
1. Add student
2. List students
3. Advance year
4. Exit
The program will display the menu, and then ask the user to choose an option. Depending on the option chosen, the program will run different method. One of the methods will ask the user to input student data,
another method will list all students in the database,
the third method will advance all students to the next year,
Lastly, the fourth method will exit the program.

Planning the Main code

We are going to create a few convenience methods. It's generally a good idea separate your code into small pieces that do one thing, instead of having one big block of code that does everything. This makes your code easier to read and maintain.

If I was writing the code, I would create the following static methods in my Main class.

  • printMenu()
  • takeOption()
  • inputStudent()
  • printStudents()
  • advanceYears()
For now, these do nothing.

I would also add a static method called scanner to the Main class. This would ensure that I only have to create one Scanner and allow me to reuse it across all my static methods.

Then I would create the simplest implementation I can think of for these methods.
So that:
The printMenu() actually prints the menu (easy!).
The takeOption() method asks the user for a single int value.
The inputStudent() method asks user for an input in sequence, but does not do anything with it.
printStudents() just displays "Work in progress" and
advanceYears() does nothing. And I would create an infinite loop in my main method, it would call the printMenu and takeOption() methods, and depending on the options selected call one of the other methods.

import java.util.Scanner;

public class Main
{
    public static Scanner scanner;

    public static void main(String[] args)
    {
        scanner = new Scanner(System.in);

        while (true) {
            printMenu();
            String option = takeOption();
            if (option.equalsIgnoreCase("1"))
            {
                inputStudent();
            }
            else if (option.equalsIgnoreCase("2"))
            {
                printStudents();
            }
            else if (option.equalsIgnoreCase("3"))
            {
                advanceYears();
            }
            else if (option.equalsIgnoreCase("4"))
            {
                break; // This "breaks" the loop, forcing it to stop
            }
        }
    }

    public static void printMenu()
    {
        System.out.println("== MENU ==");
        System.out.println("1) Add student");
        System.out.println("2) Advance year");
        System.out.println("3) Display data");
        System.out.println("4) Exit");
    }

    /**
     * Separate method in case we want to validate the input in the future
     */
    public static String takeOption()
    {
        return scanner.nextLine();
    }

    public static void inputStudent()
    {
        System.out.println("Enter name");
        String name = scanner.nextLine();

        System.out.println("Enter age");
        String age = scanner.nextLine();

        System.out.println("Enter course");
        String course = scanner.nextLine();

        System.out.println("Enter year");
        String year = scanner.nextLine();
    }

    public static void printStudents()
    {
        System.out.println("Work in progress");
    }

    public static void advanceYears()
    {}
}
